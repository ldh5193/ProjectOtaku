서버나 클라우드 비용 발생을 원천 차단하고, 1인 개발자로서 가장 빠르고 가볍게 시작할 수 있는 **"Static Web(정적 웹) 기반 오덕로드"** 계획안입니다.

이 방식은 별도의 백엔드 서버 없이 **데이터를 파일(JSON) 형태로 관리**하며, 무료 호스팅 서비스를 활용해 운영비 **0원**을 달성하는 것이 핵심입니다.

# 🌐 오덕로드(Otaku-Road) 프로젝트 기획서 (Web 버전)

## 1. 프로젝트 개요

* **목표:** 설치가 필요 없는 웹 기반의 오프라인 굿즈샵 지도 서비스.
* **핵심 전략:** 백엔드 서버 없이 **정적 데이터(JSON)**와 **클라이언트 사이드 로직**만으로 구동하여 유지비 0원 및 관리 최소화 구현.

---

## 2. 주요 기능 (Web 기반)

* **PWA(Progressive Web App):** 웹이지만 스마트폰 홈 화면에 아이콘 추가 가능, 앱처럼 실행.
* **JSON 기반 지도 마커:** 고정된 매장 정보를 JSON 파일에서 읽어와 지도에 표시.
* **클라이언트 필터링:** 브라우저 메모리 내에서 장르/타입별 즉시 필터링.
* **외부 지도 딥링크:** 상세 정보에서 클릭 시 네이버/카카오 지도 앱으로 좌표 전달.
* **Github 제보 시스템:** 유저가 '수정 제보' 버튼을 누르면 이메일이나 Github Issue로 연결하여 수동 업데이트(DB 비용 절감).

---

## 3. 기술 스택 (Zero-Cost Stack)

| 구분 | 기술 스택 | 선정 이유 |
| --- | --- | --- |
| **Frontend** | **Next.js / React** | 컴포넌트 기반 개발 및 SEO 최적화 용이 |
| **Styling** | **Tailwind CSS** | 빠르고 직관적인 UI 디자인 및 모바일 대응 |
| **Data** | **Static JSON Files** | DB 서버 없이 Git 저장소에서 데이터 관리 |
| **Map SDK** | **Kakao/Naver Map API** | 웹용 무료 쿼터가 매우 넉넉함 (일 30만 건) |
| **Hosting** | **Vercel / GitHub Pages** | 무료 호스팅 제공 및 자동 배포 시스템 |

---

## 4. 서비스 아키텍처 (Serverless Static)

```text
[ User Browser ]
       │
       ▼ (HTTPS 요청)
[ Vercel / Cloudflare Pages (무료 호스팅) ]
       │
       ├─▶ [ Web Assets (HTML/JS/CSS) ]
       └─▶ [ Data (stores.json, brands.json) ]
       │
[ Client-side Integration ]
       │
       ├─▶ [ Kakao/Naver Map JS SDK ] (지도 렌더링)
       └─▶ [ LocalStorage ] (즐겨찾기, 최근 검색어 저장)

```

## 5. 애자일 로드맵 (Web MVP)

### **Milestone 1: 정적 지도 서비스 (1~2주)**

* **핵심:** 지도 위에 매장 마커 찍기.
* **내용:** `stores.json` 파일에 매장 좌표 입력, 지도 API 연동, 마커 클릭 시 정보창 노출.
* **배포:** Vercel을 통한 무료 웹 호스팅 시작.

### **Milestone 2: 카테고리 및 검색 (2주)**

* **핵심:** 탐색 기능 고도화.
* **내용:** 상단 필터 버튼(가챠/피규어/굿즈), 지역별 리스트 보기, 검색창 구현.
* **최적화:** 모바일 반응형 레이아웃 완성.

### **Milestone 3: 데이터 관리 체계 구축 (3~4주)**

* **핵심:** 지속 가능한 정보 업데이트.
* **내용:** **오라클 클라우드(무료)**에서 스크래퍼를 돌려 `stores.json` 자동 갱신 및 Git Push 자동화 로직 구현.
* **참여:** 구글 폼(Google Forms) 연동을 통한 유저 제보 수집 창구 마련.

### **Milestone 4: 유저 편의 기능 (5주~)**

* **핵심:** 앱 같은 웹(PWA).
* **내용:** 오프라인 모드(PWA), 내 위치 기준 최단 동선 정렬, 다크모드 지원.

## 7. 서버 없이 구현 시의 현실적인 운영 팁

* **데이터 업데이트:** 유저가 앱에서 정보를 직접 바꿀 수 없으므로, 하단에 **"잘못된 정보 신고하기"** 링크를 구글 폼으로 연결하세요. 폼에 들어온 제보를 확인하여 JSON 파일만 수정하면 서비스에 즉시 반영됩니다.
* **오라클 서버 활용:** 오라클 서버에서 주기적으로 최신 정보를 긁어와서 GitHub의 JSON 파일을 자동으로 업데이트하는 스크립트를 짜두면, 백엔드 서버 없이도 정보가 최신으로 유지되는 **'가짜 백엔드'** 효과를 낼 수 있습니다.

# 서비스 발전 로드맵

Milestone 1~4(MVP)는 완료된 상태. 아래는 서비스 고도화를 위한 향후 기능 기획.

## 현황 분석

### 현재 보유 기능
- 네이버 지도 기반 매장 마커 + 인포윈도우
- 8장르 멀티 필터 (anime, figure, goods, manga, game, idol, TCG, gashapon)
- 매장명/주소 실시간 검색 (300ms 디바운스)
- 매장 상세 (영업시간, 전화, 웹사이트, 썸네일, 데이터 신선도 배지)
- 길찾기 연동 (모바일 앱/웹 자동 전환)
- 제보 시스템 (정보 수정 + 매장 추가 → GitHub Issue)
- 네이버 지도 URL 임포트로 매장 등록
- PWA 지원, 반응형 (모바일 바텀시트 / 데스크톱 사이드패널)

### 경쟁 환경
현재 한국에 인터랙티브 오타쿠 매장 지도 서비스는 사실상 없음.
- 블로그/커뮤니티 글: 정적, 검색·필터 불가, 업데이트 안 됨
- 네이버/카카오 지도: 오타쿠 장르 분류 없음
- 국전 공식 사이트: 단일 건물, 열악한 UX
- 일본 서비스 (아키하바라 가이드 등): 한국 미대응

### 차별화 기회
1. **작품/캐릭터 기반 검색** — 기존 서비스에 전무
2. **팝업스토어 트래커** — SNS에 흩어진 정보의 집약
3. **크라우드소싱 재고 목격** — 실시간 정보 공유

## Phase 5: 지도 & 탐색 강화

### 5-1. 마커 클러스터링
- 줌아웃 시 밀집 지역(홍대 20개+) 마커 겹침 방지
- 클러스터 클릭 → 해당 지역 줌인
- 클러스터에 매장 수 표시

### 5-2. "지금 영업중" 필터
- 요일별 영업시간 데이터 구조 확장 (현재 단일 문자열 → 구조화된 스케줄)
- 현재 시간 기준 "영업중 / 곧 마감 / 영업종료" 상태 표시
- 필터 토글로 영업중 매장만 표시

### 5-3. 지하철역 기반 탐색
- 주요 역(홍대입구, 동대문, 강남 등)별 매장 그룹핑
- "OO역 근처 매장" 바로가기
- 역에서 도보 시간 표시

### 5-4. 복합건물 층별 안내
- 국전, 신도림 테크노마트 등 빌딩 내 매장 층별 표시
- 건물 마커 클릭 → 층별 매장 리스트 펼침
- 데이터 구조: `floor` 필드 추가

## Phase 6: 오타쿠 특화 기능

### 6-1. 작품/캐릭터 기반 검색
- 매장별 취급 작품 태그 (예: 체인소맨, 하이큐, 주술회전)
- "체인소맨 굿즈 어디서 살 수 있어?" 검색 가능
- 태그는 크라우드소싱 + 관리자 큐레이션 병행
- 인기 작품 태그 자동완성

### 6-2. 팝업스토어 트래커
- 기간 한정 팝업매장 날짜 + 위치 등록
- 진행중 / 예정 / 종료 상태 자동 표시
- 지도에 팝업 전용 마커 스타일
- 캘린더 뷰로 일정 확인 가능

### 6-3. 재고 목격 제보
- "오늘 [매장]에서 [시리즈] 봤어요" 가벼운 제보
- 매장 상세에 최근 목격 리스트 표시
- 시간 경과에 따라 자동 페이드아웃 (3일 후 흐리게, 7일 후 삭제)

### 6-4. 가챠 머신 위치/라인업
- 매장별 가챠 머신 유무 표시
- 현재 라인업 (어떤 시리즈가 들어있는지) 크라우드소싱
- 가챠 전용 필터

### 6-5. 세부 장르 태그 확장
- 현재 8장르에 하위 태그 추가
- 상품 유형: 아크릴스탠드, 이타백, 블라인드박스, 포토카드, 트레이딩카드 등
- 브랜드 포커스: 반다이, 굿스마일, 코토부키야 등
- 필터 UI: 대분류 선택 후 세부 태그 필터

## Phase 7: 사용자 참여 & 커뮤니티

### 7-1. 한줄 리뷰/팁
- 매장별 짧은 텍스트 리뷰 (200자 제한)
- 예: "피규어 종류 다양, 공간은 좁음", "일본어 가능"
- 유용해요 투표로 상위 노출
- 비로그인 작성 가능 (IP 기반 중복 방지)

### 7-2. 매장 사진 업로드
- 외관/내부 사진 첨부 (작은 매장 찾기에 필수)
- 이미지 호스팅: GitHub Issues 또는 외부 CDN
- 매장 상세 상단 갤러리 슬라이드

### 7-3. "여기 있어요" 방문 인증
- 위치 기반 방문 체크인 (매장 반경 100m 이내)
- 체크인 수 → 데이터 신선도 자동 갱신
- 매장 상세에 "최근 N명 방문" 표시

### 7-4. 기여자 배지/레벨
- 기여 유형: 리뷰, 사진, 제보, 방문 인증, 재고 목격
- 기여도에 따라 레벨 부여 (구글 로컬가이드 방식)
- 프로필 페이지에 배지 및 기여 히스토리
- 신뢰 등급 높은 기여자의 수정은 빠른 반영

### 7-5. 데이터 모더레이션
- 사용자 제출물 리뷰 큐 (관리자 승인 후 반영)
- 신뢰 등급 높은 기여자는 자동 승인
- 신고/비추 누적 시 자동 숨김

## 인프라 로드맵

### 호스팅 전환
- 현재: Vercel 무료 티어
- 계획: Oracle Cloud Free ARM A1 (4 OCPU / 24GB RAM)
- Next.js standalone 모드 + GitHub Actions CI/CD + Caddy 리버스 프록시

### 사용자 기능을 위한 인프라
- Phase 7~8 기능(리뷰, 사진, 방문 인증 등)은 데이터 저장소 필요
- 선택지: SQLite (서버 내) / Supabase 무료 티어 / GitHub Issues API 활용
- 인증: 소셜 로그인 (카카오/네이버) 또는 익명 + IP 기반

### 비용 원칙
- 무료 유지가 최우선
