서버나 클라우드 비용 발생을 원천 차단하고, 1인 개발자로서 가장 빠르고 가볍게 시작할 수 있는 **"Static Web(정적 웹) 기반 오덕로드"** 계획안입니다.

이 방식은 별도의 백엔드 서버 없이 **데이터를 파일(JSON) 형태로 관리**하며, 무료 호스팅 서비스를 활용해 운영비 **0원**을 달성하는 것이 핵심입니다.

---

# 🌐 오덕로드(Otaku-Road) 프로젝트 기획서 (Web 버전)

## 1. 프로젝트 개요

* **목표:** 설치가 필요 없는 웹 기반의 오프라인 굿즈샵 지도 서비스.
* **핵심 전략:** 백엔드 서버 없이 **정적 데이터(JSON)**와 **클라이언트 사이드 로직**만으로 구동하여 유지비 0원 및 관리 최소화 구현.

---

## 2. 주요 기능 (Web 기반)

* **PWA(Progressive Web App):** 웹이지만 스마트폰 홈 화면에 아이콘 추가 가능, 앱처럼 실행.
* **JSON 기반 지도 마커:** 고정된 매장 정보를 JSON 파일에서 읽어와 지도에 표시.
* **클라이언트 필터링:** 브라우저 메모리 내에서 장르/타입별 즉시 필터링.
* **외부 지도 딥링크:** 상세 정보에서 클릭 시 네이버/카카오 지도 앱으로 좌표 전달.
* **Github 제보 시스템:** 유저가 '수정 제보' 버튼을 누르면 이메일이나 Github Issue로 연결하여 수동 업데이트(DB 비용 절감).

---

## 3. 기술 스택 (Zero-Cost Stack)

| 구분 | 기술 스택 | 선정 이유 |
| --- | --- | --- |
| **Frontend** | **Next.js / React** | 컴포넌트 기반 개발 및 SEO 최적화 용이 |
| **Styling** | **Tailwind CSS** | 빠르고 직관적인 UI 디자인 및 모바일 대응 |
| **Data** | **Static JSON Files** | DB 서버 없이 Git 저장소에서 데이터 관리 |
| **Map SDK** | **Kakao/Naver Map API** | 웹용 무료 쿼터가 매우 넉넉함 (일 30만 건) |
| **Hosting** | **Vercel / GitHub Pages** | 무료 호스팅 제공 및 자동 배포 시스템 |

---

## 4. 서비스 아키텍처 (Serverless Static)

```text
[ User Browser ]
       │
       ▼ (HTTPS 요청)
[ Vercel / Cloudflare Pages (무료 호스팅) ]
       │
       ├─▶ [ Web Assets (HTML/JS/CSS) ]
       └─▶ [ Data (stores.json, brands.json) ]
       │
[ Client-side Integration ]
       │
       ├─▶ [ Kakao/Naver Map JS SDK ] (지도 렌더링)
       └─▶ [ LocalStorage ] (즐겨찾기, 최근 검색어 저장)

```

## 5. 애자일 로드맵 (Web MVP)

### **Milestone 1: 정적 지도 서비스 (1~2주)**

* **핵심:** 지도 위에 매장 마커 찍기.
* **내용:** `stores.json` 파일에 매장 좌표 입력, 지도 API 연동, 마커 클릭 시 정보창 노출.
* **배포:** Vercel을 통한 무료 웹 호스팅 시작.

### **Milestone 2: 카테고리 및 검색 (2주)**

* **핵심:** 탐색 기능 고도화.
* **내용:** 상단 필터 버튼(가챠/피규어/굿즈), 지역별 리스트 보기, 검색창 구현.
* **최적화:** 모바일 반응형 레이아웃 완성.

### **Milestone 3: 데이터 관리 체계 구축 (3~4주)**

* **핵심:** 지속 가능한 정보 업데이트.
* **내용:** **오라클 클라우드(무료)**에서 스크래퍼를 돌려 `stores.json` 자동 갱신 및 Git Push 자동화 로직 구현.
* **참여:** 구글 폼(Google Forms) 연동을 통한 유저 제보 수집 창구 마련.

### **Milestone 4: 유저 편의 기능 (5주~)**

* **핵심:** 앱 같은 웹(PWA).
* **내용:** 오프라인 모드(PWA), 내 위치 기준 최단 동선 정렬, 다크모드 지원.

## 7. 서버 없이 구현 시의 현실적인 운영 팁

* **데이터 업데이트:** 유저가 앱에서 정보를 직접 바꿀 수 없으므로, 하단에 **"잘못된 정보 신고하기"** 링크를 구글 폼으로 연결하세요. 폼에 들어온 제보를 확인하여 JSON 파일만 수정하면 서비스에 즉시 반영됩니다.
* **오라클 서버 활용:** 오라클 서버에서 주기적으로 최신 정보를 긁어와서 GitHub의 JSON 파일을 자동으로 업데이트하는 스크립트를 짜두면, 백엔드 서버 없이도 정보가 최신으로 유지되는 **'가짜 백엔드'** 효과를 낼 수 있습니다.
